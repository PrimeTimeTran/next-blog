## System Structure

All knowledge follows this structure:

- Atom → single concept
- Pattern → reusable strategy
- Hub → domain organization
- Meta → cross-domain reasoning

See: [[Note Type System (Atom vs Pattern vs Hub vs Meta)]]

---

## Entry Points

- Start with a domain hub
- Drill down into atoms for understanding
- Use patterns for application
- Use meta concepts for evaluation

---

## Design Principles

- One idea per note
- Prefer linking over duplication
- Separate structure from content
- Keep hubs lightweight
- Keep atoms self-contained

---

## Long-Term Goal

To build a system that:

- scales across disciplines
- supports deep understanding
- enables problem solving
- can be exported into structured writing (e.g. books)

---

## Notes

This is a living system.
Structure may evolve, but clarity and modularity are preserved.
