# 🧠 Object-Oriented Programming (OOP) Hub

OOP is a programming paradigm based on organizing code into **objects** that combine:

- data (state)
- behavior (methods)

---

# 🧱 Core Principles of OOP

## [[Encapsulation]]

Encapsulation is the bundling of data and methods, restricting direct access to internal state.

- [[Getters and Setters]] — controlled access to private data
- [[Access Modifiers]] — define visibility (public/private/protected)
- [[Data Hiding]] — protect internal state from direct access

---

## [[abstraction]]

Abstraction hides implementation details and exposes only essential behavior.

- [[Abstract Classes]] — partially defined base classes
- [[Interfaces]] — contracts that define required behavior
- [[Abstract Methods]] — methods without implementation
- [[Concrete Methods]] — fully implemented methods

---

## [[Inheritance]]

Inheritance allows one class to derive properties and behavior from another.

- [[Single Inheritance]] — one parent class
- [[Multilevel Inheritance]] — chain inheritance (A → B → C)
- [[Hierarchical Inheritance]] — one parent, many children
- [[Multiple Inheritance]] — multiple parents (language-dependent)
- [[Hybrid Inheritance]] — combination of multiple types

---

## [[Polymorphism]]

Polymorphism allows the same interface to behave differently depending on context.

### Compile-time (static)

- [[Method Overloading]] — same method, different parameters
- [[Operator Overloading]] — redefine operator behavior

### Runtime (dynamic)

- [[Method Overriding]] — subclass replaces parent method
- [[Virtual Methods]] — enable dynamic dispatch

---

# 🧩 Classes and Objects

## [[Class]]

A blueprint for creating objects.

- [[Class Variables]] — shared across instances
- [[Class Methods]] — operate at class level
- [[Class Constructors]] — initialize objects
- [[Class Destructors]] — cleanup before destruction

---

## [[Objects]]

An object is an instance of a class.

- [[Object Instantiation]] — creating an object
- [[Object Communication]] — interaction via methods
- [[Object Lifecycle]] — creation → use → destruction

---

# ⚙️ Methods

- [[Instance Methods]] — operate on object state
- [[Static Methods]] — belong to class, not instance
- [[Virtual Methods]] — support runtime polymorphism

---

# 🔨 Constructors

- [[Parameterized Constructors]] — initialize with arguments
- [[Copy Constructors]] — duplicate existing objects
- [[Constructor Overloading]] — multiple constructors

# 🧠 Big Picture Summary

OOP is built on four pillars:

- Encapsulation → protects data
- Abstraction → hides complexity
- Inheritance → reuses structure
- Polymorphism → enables flexibility

Everything else (classes, objects, methods) supports these ideas.
