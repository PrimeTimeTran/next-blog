There are many file extensions used in Machine Learning

.pickle

- Used for serializing and de-serializing Python objects, useful for saving machine learning models, preprocessing steps, and other data structures.

.h5

- Used for storing large amounts of numerical data, often used with Keras for saving model weights or entire models.

.keras

- Used to save entire Keras models, including architecture, weights, and optimizer state, in a single HDF5 file.

.ckpt

- Used by TensorFlow to save and restore model checkpoints, which include model weights and state of the optimizer.

.safetensor

- Used for efficiently storing and sharing tensor data, often used in scenarios requiring high performance, such as model training and inference.

.pb

- TensorFlow's protocol buffer format used for saving entire models, which can be deployed across different platforms.

.joblib

- Similar to .pickle but optimized for large numpy arrays, often used for saving machine learning models and pipelines in scikit-learn.

.onnx

- Used for saving models in an open format supported by many frameworks, allowing models to be easily transferred and used across different machine learning tools.
